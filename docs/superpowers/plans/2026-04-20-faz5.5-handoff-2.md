## Bekleyen Sorun — Booking Hatası ❌

**Semptom:** Randevu alırken hata oluştu (bot veya kullanıcı tarafı — tam mesaj bilinmiyor).

**Olası root cause'lar (öncelik sırasıyla):**

1. **`staffId` Airtable field name uyuşmazlığı** — `Randevular` tablosundaki field adı tam `staffId` mi? (büyük/küçük harf dahil). `createAppointment`'ta `fields.staffId = [id]` yazıyor.
2. **Bot `staff_id: null` gönderdi** — `executeTool`'da `String(null) = "null"`, `"null" || undefined = "null"` (truthy), Airtable'a `["null"]` yazılıyor → hata. Fix: `toolInput.staff_id && toolInput.staff_id !== 'null' && toolInput.staff_id !== 'undefined' ? toolInput.staff_id : undefined`
3. **Airtable `staffId` Link field write formatı** — `fields.staffId = [id]` doğru format, ama `id` valid Airtable record ID olmalı (recXXX). Bot yanlış ID göndermiş olabilir.
4. **`staffName` Lookup field'a yazma girişimi** — `createAppointment`'ta `staffName` yazılmıyor (sadece `staffId`), bu OK.

**Debug adımı (yeni sessionda ilk iş):**
```
# Dev server console'da şu logu ara:
[book_appointment] Airtable error: ...
```
Veya Airtable'da son başarısız request'i kontrol et.

**Hızlı fix (root cause 2 için, güvenli taraf):**
`app/api/chat/route.ts` içindeki her `toolInput.staff_id || undefined` satırını şununla değiştir:
```ts
const staffId = (toolInput.staff_id && toolInput.staff_id !== 'null' && toolInput.staff_id !== 'undefined')
  ? toolInput.staff_id : undefined;
```
